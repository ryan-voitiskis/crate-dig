Review existing genre tags against Discogs, Beatport, and audio evidence. The agent identifies BPM-implausible tags and evidence conflicts, presenting only the tracks that need attention. Every change requires your explicit approval.

**Paste into your agent to start:**

```
Audit my existing genre tags.
```

<Aside type="caution">
  No auto-tagging. Every genre change requires your explicit approval before it is staged or exported.
</Aside>

## When to audit

- After a bulk genre classification pass — catch any misclassifications
- When you suspect a genre was applied too broadly (e.g. everything tagged "Techno")
- After importing tracks from another collection with different tagging conventions
- Periodically, as enrichment data improves over time

## How it works

The `audit_genres` tool applies the same genre decision tree as classification, but compares the evidence-based result against the track's existing genre tag. Only conflicts and manual-review tracks are shown — confirmed tracks are counted in the summary but omitted from results.

## Audit vs Classification

| | Classification | Audit |
|---|---|---|
| **Tool** | `classify_tracks` | `audit_genres` |
| **Scope** | Ungenred tracks (enforces `has_genre=false`) | Tracks with genres (enforces `has_genre=true`) |
| **Goal** | Assign genres from scratch | Verify existing genres are correct |
| **Output** | All results grouped by confidence | Only conflicts and manual-review tracks |
| **Typical volume** | All ungenred tracks | Subset with conflicts only |

## Cache coverage

All providers must be at 100% before starting. Run `cache_coverage(has_genre=true)` to check. If any provider is below 100%, hydrate first.

<Aside type="tip">
  Run audio analysis first (local, fast), then enrichment (external, rate-limited). This order minimizes wall-clock time.
</Aside>
