If your library has missing genres, wrong labels, inconsistent naming, or sparse metadata, you might not know where to start. Library Cleanup breaks the work into five explicit sessions, each with clear entry and exit criteria.

<Aside type="danger">
  **Changes are staged in memory and exported as XML.** The Rekordbox database only updates after you import the XML file. Each session reads from the database, so you **must** import the XML before starting the next session.
</Aside>

## Why order matters

The dependency chain looks like this:

**Clean metadata → accurate enrichment matches → label data → genre signals**

- **Enrichment** matches on artist name and track title. Typos and inconsistent naming cause missed or wrong matches.
- **Genre classification** uses enrichment data (Discogs styles, Beatport genres), labels, and audio analysis as evidence. The more signals available, the more accurate the result.
- **Genre audit** verifies existing tags against the same cached evidence. Running it before enrichment means you're checking tags without the data needed to correct them.

Working out of order means you'll either get poor results or end up re-running workflows.

## Sessions

### Session 1: Collection Audit

<Badge text="Entry" variant="note" /> MCP server connected.

**Paste into your agent to start this session:**

```
Audit my collection for naming and tagging issues.
```

The agent will follow the Collection Audit SOP — fix artist names, track titles, and naming conventions. Enrichment matches on these fields, so cleaning them first dramatically improves match rates.

<LinkCard title="Collection Audit workflow" href="/workflows/collection-audit/" />

#### CHECKPOINT

```
preview_changes()
write_xml()
```

Import the XML file into Rekordbox. Verify metadata is clean by spot-checking corrected tracks in Rekordbox.

<Aside type="tip">
  This is a natural context-reset point. You can close this agent session and start a new one for Session 2.
</Aside>

### Session 2: Enrichment

<Badge text="Entry" variant="note" /> Clean metadata — Session 1 complete and XML imported into Rekordbox.

**Paste into your agent to start this session:**

```
Check my cache coverage and hydrate any gaps in enrichment and audio analysis.
```

**Work:** Hydrate all caches. For large collections (1000+ tracks), use the CLI — it runs unattended with progress bars:

```sh
reklawdbox hydrate --cpu overnight -y
```

This runs Discogs, Beatport, and audio analysis in one pass. The `--cpu overnight` flag uses max cores without nicing; `-y` skips the confirmation prompt. Use filters like `--artist`, `--genre`, or `--added-after` to scope to a subset if needed.

Alternatively, use MCP tools in batches:

```
analyze_audio_batch(skip_cached=true, max_tracks=200)
enrich_tracks(skip_cached=true, providers=["discogs"], max_tracks=50)
enrich_tracks(skip_cached=true, providers=["beatport"], max_tracks=50)
```

**Exit criteria:**

```
cache_coverage()
```

All providers must be at 100%.

<Aside type="tip">
  This is a natural context-reset point. No XML export is needed — enrichment data is persisted in the local cache database.
</Aside>

---

### Session 3: Label Verification

<Badge text="Entry" variant="note" /> 100% cache coverage for all providers.

**Paste into your agent to start this session:**

```
Find tracks missing labels in my collection and fill them from enrichment data.
```

Check label coverage across your collection:

```
search_tracks(has_label=false)
```

Labels are the strongest genre signal for electronic music. Review unlabelled tracks — enrichment usually has label data. Where gaps remain, use `write_file_tags` to write labels directly to audio files:

```
write_file_tags(writes=[{"path": "...", "tags": {"publisher": "Label Name"}}])
```

<Aside type="note">
  Labels are stored in the `publisher` tag field. After writing, Rekordbox will pick up changes on its next file analysis. For large batches, consider using the Rekordbox "Reload Tags" function.
</Aside>

#### CHECKPOINT

No XML export needed for label changes — they're written directly to audio files. Verify in Rekordbox that labels appear.

<Aside type="tip">
  This is a natural context-reset point. You can close this agent session and start a new one for Session 4.
</Aside>

---

### Session 4: Genre Classification

<Badge text="Entry" variant="note" /> Labels verified, 100% cache coverage.

**Paste into your agent to start this session:**

```
Classify genres for my ungenred tracks.
```

1. **Normalize aliases:**

   ```
   suggest_normalizations()
   ```

   Stage non-debatable fixes (e.g. `Hip-Hop` → `Hip Hop`, `DnB` → `Drum & Bass`):

   ```
   update_tracks(changes=[{"track_id": "...", "genre": "..."}])
   ```

2. **Classify ungenred tracks:**

   ```
   classify_tracks(max_tracks=50)
   ```

   The server applies the genre decision tree using all cached evidence. Review results grouped by confidence:

   - **High/Medium:** approve in bulk or per-track
   - **Low/Insufficient (`needs_review`):** agent reviews with artist/title context, presents recommendations

   Stage approved classifications:

   ```
   update_tracks(changes=[{"track_id": "...", "genre": "..."}])
   ```

   Repeat with `offset` pagination until all tracks are processed.

<LinkCard title="Genre Classification workflow" href="/workflows/genre-classification/" />

#### CHECKPOINT

```
preview_changes()
write_xml()
```

Import the XML file into Rekordbox. Verify genres by browsing the genre column in Rekordbox.

<Aside type="tip">
  This is a natural context-reset point. You can close this agent session and start a new one for Session 5.
</Aside>

### Session 5: Genre Audit

<Badge text="Entry" variant="note" /> Genres classified, XML imported into Rekordbox.

**Paste into your agent to start this session:**

```
Audit my existing genre tags.
```

1. **Audit existing genres:**

   ```
   audit_genres(max_tracks=50)
   ```

   The summary shows confirmed vs conflict counts. Only conflicts and manual-review tracks appear in results.

2. **Review conflicts** and approve or reject changes:

   ```
   update_tracks(changes=[{"track_id": "...", "genre": "..."}])
   ```

   Repeat with `offset` pagination until all tracks are processed.

<LinkCard title="Genre Audit workflow" href="/workflows/genre-audit/" />

#### CHECKPOINT

```
preview_changes()
write_xml()
```

Import the XML file into Rekordbox. Spot-check corrected genres.

## Skipping sessions

Not every library needs all sessions:

- **Metadata already clean?** Skip Session 1, start at Session 2 (Enrichment).
- **Already enriched?** Check with `cache_coverage()` — if 100%, jump to Session 3.
- **Labels already present?** Skip Session 3.
- **No existing genre tags?** Skip Session 5 — classification covers untagged tracks.
- **No ungenred tracks?** Skip Session 4 — audit covers tagged tracks.
