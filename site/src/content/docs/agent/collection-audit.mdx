---
title: Collection Audit
description: Agent SOP for detecting and fixing naming, tagging, and convention violations.
---

Scan audio files for convention violations and fix them with user approval.

## Constraints

- **Never rename imported files.** Before any rename, check `search_tracks(path="...")` â€” if the file is in Rekordbox, defer the rename. Rekordbox cannot update paths via XML import (creates duplicates). The user must manually relocate via Rekordbox.
- **Never manually resolve with `resolution="fixed"`** â€” that value is reserved for the scan engine's auto-resolution.
- Resolution options: `accepted_as_is`, `deferred`, `wont_fix`.

## Prerequisites

- reklawdbox MCP server connected
- No external tools required â€” all tag operations use `audit_state` and `write_file_tags`

## Steps

### 0. Scope and issue type selection

Ask user:
1. **Scope** â€” entire collection, specific artist dir, album, play subdirectory, or custom path
2. **Issue types** â€” present full list. Specifically ask: "Skip GENRE_SET? It can produce thousands of flags."

Record choices as `skip_issue_types` for the scan.

### 1. Scan

```
audit_state(scan, scope="/path/", skip_issue_types=["GENRE_SET"])
```

Review summary: `files_in_scope`, `new_issues` by type, `total_open`.

If `total_open` = 0 â†’ skip to Final Report.

### 2. Review and triage

Query issues by type:

```
audit_state(query_issues, scope="/path/", status="open", issue_type="WAV_TAG3_MISSING")
```

Categorize by safety tier:

- **Safe** (auto-fixable): `WAV_TAG3_MISSING`, `WAV_TAG_DRIFT`, `ARTIST_IN_TITLE` â†’ Step 3
- **Rename-safe**: `ORIGINAL_MIX_SUFFIX`, `TECH_SPECS_IN_DIR` â†’ Step 3 (if not imported)
- **Review**: all others â†’ Step 4

Present triage summary to user.

Ask user: "Start with safe auto-fixes?"

### 3. Fix safe issues

Present planned writes for approval, then apply:

```
write_file_tags(writes=[{path: "...", tags: {artist: "...", title: "..."}}])
```

Re-scan to verify fixes (scan auto-marks resolved issues as `fixed`):

```
audit_state(scan, scope="/path/")
```

Never manually resolve with `resolution="fixed"` â€” that's reserved for the scan engine.

### 4. Review-tier issues

Present each issue type in batches:

**Empty/missing tags** (`EMPTY_ARTIST`, `EMPTY_TITLE`, `MISSING_TRACK_NUM`, `MISSING_ALBUM`, `MISSING_YEAR`):
- Infer from filename/directory structure
- Use `lookup_discogs()` / `lookup_beatport()` for missing album/year
- Ask user to confirm each fix

**Filename issues** (`BAD_FILENAME`, `MISSING_YEAR_IN_DIR`, `FILENAME_TAG_DRIFT`):
- For `FILENAME_TAG_DRIFT`: compare `detail.filename` vs `detail.tag` values. If differences are only character substitutions (`?`â†’`_`, `/`â†’`-`, `:`â†’`-`), the tag is authoritative â€” resolve as `accepted_as_is`. If real drift, use `lookup_discogs()` / `lookup_beatport()` to determine correct value.
- For imported files needing rename: defer with note about manual Rekordbox relocate

**No-tag files** (`NO_TAGS`):
- Infer metadata from parent directory name, filename, and companion files
- Present inferred values to user for confirmation before writing

**Genre tags** (`GENRE_SET`, if included):
- Per file: keep / clear / migrate to comments
- When migrating: read existing comment first, prepend genre, never overwrite

For items user wants to skip:

```
audit_state(resolve_issues, issue_ids=[...], resolution="accepted_as_is", note="...")
audit_state(resolve_issues, issue_ids=[...], resolution="deferred", note="...")
audit_state(resolve_issues, issue_ids=[...], resolution="wont_fix", note="...")
```

### 5. Verification scan

```
audit_state(scan, scope="/path/")
```

If `total_open` > 0, return to Step 2 or confirm remaining items are intentionally deferred.

### 6. Final report

```
audit_state(get_summary, scope="/path/")
```

Present: scope, files scanned, pass rate, fixes by type, deferred items, next steps (Rekordbox import, genre classification SOP).
