These pages are designed for agent consumption. Each SOP is a concise, imperative
step-by-step guide with tool calls, decision trees, and explicit user checkpoints.

Each page includes an **Init** block — a copyable prompt you can paste into your
agent session to start the workflow.

### LLM-friendly access

All Agent SOPs are available as plaintext markdown at
[`/_llms-txt/agent-sops.txt`](/_llms-txt/agent-sops.txt) — use this URL in
Init prompts so agents can fetch the complete SOP without HTML parsing.

See also [`/llms.txt`](/llms.txt) for the full site index.

## Workflows

- [Batch Import](/agent/batch-import/) — prepare newly acquired music for Rekordbox import
- [Genre Classification](/agent/genre-classification/) — classify genres using Discogs, Beatport, and audio evidence
- [Set Building](/agent/set-building/) — build transition-scored DJ set sequences
- [Collection Audit](/agent/collection-audit/) — detect and fix naming, tagging, and convention violations

## Before You Start

Run `cache_coverage()` to check enrichment and analysis readiness. Most workflows
require cached data — enrichment and audio analysis should be run first if coverage
is low.

## Key Principles

- **Read-only DB.** All changes are staged in memory and exported as XML.
- **Human approval required.** Never auto-apply changes. Present recommendations and wait for confirmation.
- **Cache-first.** Workflows run from cached data. Flag gaps rather than triggering external calls mid-workflow.
- **Export path:** `update_tracks()` → `preview_changes()` → `write_xml()`
