### Step A: Gather mapped genres

1. From `discogs_mapped_genres`, collect `genre` values, deduplicate â†’ `discogs_genres` (set).
2. `beatport_mapped_genre` â†’ `beatport_genre` (single value or null).

### Step A.2: BPM plausibility check

For each mapped genre that has a `bpm_range` in the classification output, compare the track's BPM (top-level `bpm` field) against the range:

- If `rekordbox_bpm` falls **outside** the typical range, mark that source's suggestion as **bpm-implausible**. - If `rekordbox_bpm` falls **within** the range, or the genre has no `bpm_range` (null), proceed normally.

Also check `current_genre_bpm_range` against the track's `bpm`. If the current genre is bpm-implausible, note it alongside genre results â€” this is informational for conflict review.

### Step A.3: Label-inferred genre

If `label_genre` is non-null:

1. Add `label_genre` to the candidate set alongside Discogs and Beatport genres.
2. **Weight:** Label inference is high-confidence evidence â€” comparable to a Beatport exact match.
3. **Agreement:** If `label_genre` matches any Discogs or Beatport genre, boost that genre's confidence by one level (lowâ†’medium, mediumâ†’high). Note "label confirms".
4. **Disagreement:** If `label_genre` conflicts with all other sources:

5. **Solo signal:** If `label_genre` is the only genre evidence (no Discogs/Beatport mapped genres), treat as confidence = **medium**, note "label-only inference".

### Step C: Find consensus

Only reached if no audio veto fired.

4. **Full consensus:** All non-null sources (Discogs, Beatport, label) agree â†’ confidence = **high**.

5. **Partial consensus (Discogs only):** `beatport_genre` is null, `discogs_genres` has exactly one entry â†’ confidence = **medium**.

6. **Partial consensus (Beatport only):** `discogs_genres` is empty, `beatport_genre` is non-null â†’ confidence = **medium**.

### Step C2: Note secondary influences

- Format: present influences alongside the genre suggestion, e.g. "Techno (influences: IDM)".
- Influences are informational â€” they do not affect confidence or the primary genre.

## BPM Mismatch Detection

Side-check during evidence evaluation and review. When `bpm_agreement` is false:

1. Compute `ratio = stratum_bpm / rekordbox_bpm`.
2. If ratio is near 2.0 or 0.5 (within Â±15%), it is a half-time/double-time disagreement. Use `rekordbox_bpm` for all BPM thresholds in Steps Bâ€“D â€” Rekordbox's analyzer has been validated by the user via beatgridding.
3. If ratio is not near a 2:1 relationship, flag as a genuine BPM conflict. Present both values to the user alongside genre results.

BPM mismatches affect audio vetoes (B.3 `slow`/`fast` flags) and subgenre inference (D.2 BPM table). An incorrect BPM can shift a track between Downtempo and Techno ranges.
