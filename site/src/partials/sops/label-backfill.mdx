## Constraints

- **Enrichment-dependent.** Labels come from Discogs enrichment cache. Tracks without cached enrichment are skipped, not fetched.
- **No auto-tagging without review.** The backfill tool stages non-conflicting labels automatically, but conflicts require human approval before staging.
- **XML export only.** All changes flow through `update_tracks` → `preview_changes` → `write_xml`.
- **Self-released tracks are skipped.** Discogs "Not On Label" entries are filtered out — they carry no useful signal.

## Prerequisites

```
cache_coverage(has_label=false)
```

Check Discogs coverage for unlabeled tracks. If below 80%, hydrate first:

```
enrich_tracks(has_label=false, skip_cached=true, providers=["discogs"], max_tracks=50)
```

Repeat until Discogs coverage reaches 80%+. Report progress between batches.

## Steps

### 1. Backfill labels

```
backfill_labels()
```

The tool scans the entire collection (excluding samples), looks up each track's Discogs enrichment, and:
- **Fills** empty labels from enrichment (auto-staged)
- **Skips** tracks where current label matches enrichment
- **Reports conflicts** where current label differs from enrichment (not staged)

Report the summary: "Backfill complete: N labels staged, N already correct, N conflicts, N without enrichment."

### 2. Resolve conflicts

Group conflicts by pattern for efficient review:

#### Artist-name-as-label vs actual label

When the current label is the artist's name and Discogs has an actual label, suggest the Discogs label — it provides a stronger signal than the artist name.

```
Artist-name-as-label conflicts (N):
 #1  Fantastic Man — Antiboudi | Fantastic Man → Mule Musiq
 #2  Fantastic Man — Diaspora | Fantastic Man → Mule Musiq
 ...
```

Ask user: **"Approve all, or reject/change specific numbers."**

Stage approved changes via `update_tracks`.

#### Label name variations

When both labels refer to the same entity with different formatting (e.g. "Palette" vs "Palette Recordings", "Loma Vista" vs "Loma Vista Recordings"), suggest keeping the current label — it was set intentionally.

Present these for confirmation but recommend no change.

#### Genuine disagreements

When the Discogs label is a different entity entirely (e.g. different pressing, compilation, or wrong match), present individually with context:

```
 #N  Artist — Title | Current: Label A → Discogs: Label B
```

Ask user to pick one, or skip.

Stage approved changes via `update_tracks`.

### 3. Export

```
preview_changes()
```

Ask user: "Export these changes to XML?"

```
write_xml()
```

Report output path. Remind user to import into Rekordbox.
